# Repository Structure

This document outlines the key directories/files so reviewers can find code,
configs, and outputs quickly.

- `lab/`
  - `src/` — experiment harness, ablation batteries, utilities
  - `configs/` — JSON configs for H1/H5/H6 and variants
  - `data/` — tokenizer-aware corpora & splits (small, hashed)
  - `runs/` — hashed output directories (`config.json`, metrics, artifacts)
- `scripts/`
  - CLI helpers for data prep, post-processing, verification
  - `verify_manifest.py` ensures `reports/RESULTS_MANIFEST.json` is consistent
- `reports/`
  - CSV/JSON exports generated by `make postprocess`
  - Indexed by `reports/RESULTS_MANIFEST.json`
- `paper/`
  - LaTeX manuscript, figures, references
- `docs/`
  - REPLICATION checklist, STRUCTURE (this file), manifest usage
- `notebooks/`
  - `results_summary.ipynb` quick viewer for manifest outputs
- `Makefile`
  - `make postprocess` regenerates reports (summaries, rankings, OV tables, manifest)
- `pyproject.toml`
  - project metadata, dependencies, Black/Ruff configuration
- `.github/workflows/`
  - CI pipeline (lint, smoke test, manifest rebuild)

Useful commands:

- `python smoke_test.py` — quick environment check
- `make postprocess` — regenerate report artifacts
- `python scripts/verify_manifest.py` — validate `reports/RESULTS_MANIFEST.json`
- `pre-commit install` (once) then `pre-commit run --all-files`

For more detail on reproducing experiments, see `docs/REPLICATION.md`.
